# Marp Rules 使用ガイド

このディレクトリには、Marpスライド生成のための11種類のルールファイルが含まれています。

## ファイル一覧

### slidemarprules-1.mdc（基本）
- Marpテンプレート更新ルール
- テンプレートの使用方法、文字数制限、レイアウト制限
- **全てのプレゼンで必須**

### rules-2.mdc
- カラースキームとデザイン統一ルール
- カラーパレット、コントラスト比、ブランドガイドライン
- デザインの一貫性を保つ

### rules-3.mdc
- コードブロックとシンタックスハイライトルール
- 言語指定、フォーマット、シンタックスハイライト
- 技術プレゼンテーションに必須

### rules-4.mdc
- 表とデータ表示ルール
- 表のサイズ制限、スタイリング、データの視覚化
- データ分析・レポート系に必須

### rules-5.mdc
- アニメーションと遷移効果ルール
- CSSアニメーション、スライド遷移、アクセシビリティ配慮
- デザイン重視のプレゼンに推奨

### rules-6.mdc
- アイコンと図形使用ルール
- アイコンの選択、サイズ、図形の活用方法
- 視覚的な説明が必要な場合に有効

### rules-7.mdc
- フォントとタイポグラフィルール
- フォントファミリー、サイズ、行間、日本語対応
- 可読性重視のプレゼンに必須

### rules-8.mdc
- レイアウトと配置ルール
- グリッドシステム、余白、レイアウトパターン
- **ほぼ全てのプレゼンで推奨**

### rules-9.mdc
- 引用と参照表記ルール
- 引用の表記、参照文献、脚注、著作権
- 学術発表やデータ分析に必須

### rules-10.mdc
- アクセシビリティとユニバーサルデザインルール
- WCAG準拠、色覚異常対応、スクリーンリーダー対応
- 公的機関や大規模イベントに推奨

### rules-11.mdc
- プレゼンテーション構成とストーリーテリングルール
- 三部構成、ストーリーアーク、エンゲージメント技法
- **説得力のあるプレゼンに必須**

## 実践的な使い方

### Cursorでの設定方法

1. **AIチャットで@rulesを使用**
   ```
   例: @slidemarprules-1 @rules-2 @rules-8 @rules-11
   ```

2. **プロジェクトごとに使用するrulesを決定**
   - プレゼンの目的を明確化
   - ターゲット聴衆を特定
   - 必要なrulesを選択

3. **よく使う組み合わせをメモ**
   - プロジェクトごとに記録
   - チーム内で共有

### 段階的導入のすすめ

#### ステップ1: 基本セット（初心者向け）
```
@slidemarprules-1
@rules-8
@rules-11
```
**理由:** テンプレート、レイアウト、構成の基本を押さえる

#### ステップ2: デザイン強化
```
基本セット
+ @rules-2
+ @rules-7
```
**理由:** 色使いとフォントでプロフェッショナルな見た目に

#### ステップ3: 用途別カスタマイズ
```
ステップ2のセット
+ 用途に応じて rules-3〜6, 9, 10 を追加
```
**理由:** プレゼンの目的に特化した機能を追加

## ルール適用時の注意事項

### 1. 競合するルールに注意
- 複数のルールで同じスタイルを定義している場合、後から読み込まれたルールが優先されます
- 意図しない上書きを避けるため、必要最小限のrulesを選択してください

### 2. パフォーマンスへの配慮
- あまりに多くのrulesを同時に適用すると、処理が重くなる可能性があります
- 通常は5-7個程度のrulesを組み合わせるのが最適です

### 3. 一貫性の維持
- プロジェクト内では同じrulesの組み合わせを使用してください
- スライドごとに異なるrulesを使うと、デザインの統一感が失われます

## カスタマイズのヒント

### ルールの部分的な適用
特定のスライドでのみルールを適用したい場合:

```markdown
<!-- rules-5のアニメーションを特定スライドでのみ使用 -->
<style scoped>
/* rules-5.mdcのアニメーションをここにコピー */
</style>
```

### ルールの上書き
既存のルールを部分的に変更したい場合:

```markdown
<style>
/* rules-2のカラーを上書き */
:root {
  --primary: #YOUR_COLOR;
}
</style>
```

## トラブルシューティング

### スタイルが適用されない
1. `@rules`の指定が正しいか確認
2. ファイル名のタイプミスがないか確認
3. Cursorを再起動してみる

### デザインが崩れる
1. 適用しているrulesの数を減らしてみる
2. 競合している可能性のあるrulesを一つずつ無効化
3. テンプレート（YYYYMMDD_template.md）の設定を確認

### パフォーマンスが悪い
1. アニメーション系ルール（rules-5）を無効化
2. 複雑なレイアウトを簡素化
3. 画像サイズを最適化

## まとめ

### 最も汎用的な組み合わせ
```
@slidemarprules-1
@rules-2
@rules-8
@rules-11
```
この4つで、ほとんどのビジネスプレゼンテーションに対応できます。

### 用途に応じて追加
- **コードを含む** → `@rules-3`
- **データを含む** → `@rules-4`
- **デザイン重視** → `@rules-5` + `@rules-6` + `@rules-7`
- **引用が必要** → `@rules-9`
- **アクセシビリティ重視** → `@rules-10`
